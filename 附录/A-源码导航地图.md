# 附录 A：架构导航地图

本附录为 Claude Code 系统架构的概念性导航，帮助读者从宏观层面理解各功能模块的职责与协作关系。

---

## A.1 核心模块索引表

| 模块名称 | 职责描述 |
|---------|---------|
| CLI 入口与 REPL | 命令行入口点，负责参数解析、REPL 交互循环的渲染与会话初始化 |
| 对话主循环 | 基于 turn 的对话循环，管理消息收发与工具调度的核心调度器 |
| 查询引擎 | 封装 API 调用、流式响应处理、重试逻辑的底层引擎 |
| 工具类型系统 | 工具的泛型接口定义、执行上下文与工具工厂函数 |
| 工具注册表 | 全局工具注册、发现与组装，负责将内置工具与动态工具合并 |
| 子智能体系统 | 子智能体的生成、恢复、分叉（fork）及内置智能体定义 |
| Shell 执行引擎 | Shell 命令的权限校验、只读检测与安全沙箱执行 |
| 上下文压缩 | auto-compact、micro-compact、session memory 等多层压缩策略 |
| 钩子系统 | Pre/Post tool use hooks、session hooks 及异步钩子的注册与执行 |
| 设置系统 | 全局/项目/本地三级配置、权限规则定义与 schema 验证 |
| 记忆系统 | CLAUDE.md 发现、嵌套记忆、团队记忆与相关性匹配 |
| 技能系统 | 内置技能管理、动态加载与 slash command 注册 |
| MCP 集成 | MCP 连接管理、协议适配、资源读写与权限通道 |
| IDE 桥接 | VSCode/JetBrains 双向通信、JWT 认证与远程会话管理 |
| 协调器模式 | 多智能体协调场景下的 worker 分配与结果汇总 |
| 状态管理 | 全局应用状态 store、React 集成与 selector 模式 |

## A.2 模块依赖关系

Claude Code 的核心依赖链如下（自上而下）：

```
CLI 入口层                         参数解析 + REPL 初始化
  |
  +-- 对话主循环                    turn-based 消息循环
  |     |
  |     +-- 查询引擎               API 调用 + 流式响应
  |     |
  |     +-- 工具注册表              工具发现与路由
  |           |
  |           +-- 工具类型系统      工具接口与工厂
  |           |
  |           +-- 各工具实现        按功能域划分
  |                 |
  |                 +-- 子智能体    （递归调用对话主循环）
  |                 +-- Shell 执行
  |                 +-- 文件编辑
  |                 +-- ...
  |
  +-- 上下文压缩服务               （被对话主循环调用）
  |
  +-- 钩子系统                     （拦截工具调用前后）
  |
  +-- 全局状态                     （AppState store）
  |
  +-- 记忆系统                     （CLAUDE.md 注入）
  |
  +-- 技能系统                     （技能调用）
  |
  +-- MCP 服务                     （动态工具注入）
  |
  +-- IDE 桥接                     （外部通信）
  |
  +-- 多智能体协调模式             （协调器模式）
```

**核心循环**：CLI 入口初始化 REPL -> 用户输入触发对话主循环的 turn 循环 -> 查询引擎发起 API 调用 -> 模型返回 tool_use -> 工具注册表路由到具体工具 -> 工具执行后结果回填 -> 再次调用 API，直到模型输出结束（stop reason = "end_turn"）。

**横切关注点**：
- **钩子系统**（hooks）在工具调用前后插入拦截逻辑
- **权限系统**通过统一的权限检查函数决定是否允许执行
- **压缩系统**在上下文接近 token 上限时自动触发
- **记忆系统**在每轮开始时扫描 CLAUDE.md 并注入系统提示

## A.3 数据流路径速查

### 标准工具调用流程

```
[1] 用户输入
     |
     v
[2] 对话主循环：processUserMessage()
     |  - 追加用户消息到 messages[]
     |  - 注入 CLAUDE.md 记忆内容
     v
[3] 查询引擎：queryAPI()
     |  - 组装 system prompt + messages + tools
     |  - 调用 Anthropic Messages API（流式）
     v
[4] 流式响应处理
     |  - 逐步接收 text 和 tool_use blocks
     |  - 渲染到 REPL 终端
     v
[5] 工具路由
     |  - 按 tool_use block 中的 name 查找工具
     |  - 执行权限检查
     |  - 若需用户确认，弹出权限对话框
     v
[6] 工具执行
     |  - 调用工具的执行方法
     |  - 返回工具结果
     v
[7] 结果回填
     |  - tool_result 追加到 messages[]
     |  - 大结果持久化到磁盘
     v
[8] 循环回到 [3]
     |  - 携带完整的 messages[]（含工具结果）再次调用 API
     |  - 直到 stop_reason === "end_turn"
     v
[9] turn 结束
        - 输出最终文本响应
        - 检查是否需要 auto-compact
```

### 权限判定路径

```
工具调用请求
     |
     v
输入合法性校验
     |
     v
工具级权限逻辑检查
     |
     v
PreToolUse 钩子          -- 用户自定义钩子
     |
     v
通用权限系统检查
     |  - 检查 alwaysAllow / alwaysDeny / alwaysAsk 规则
     |  - 检查 permission mode（default / plan / auto）
     v
[执行 / 拒绝]
     |
     v
PostToolUse 钩子         -- 执行后钩子
```

### 上下文压缩触发路径

```
对话主循环 turn 结束
     |
     v
检查 token 使用量
     |
     v（接近阈值）
自动压缩触发
     |
     v
压缩服务
     |  - 分组历史消息
     |  - 调用 API 生成摘要
     |  - 替换原始消息为摘要
     v
压缩后清理
```
